# 🚀 Izra AI: Deployment & Play Store Publishing Roadmap

This guide outlines the best, most scalable approach to deploying **Izra AI** as a live website and publishing it as a native Android app on the Google Play Store.

---

## 🌐 Phase 1: Web Deployment (The Scalable Way)

For a platform using Next.js, FastAPI, and Firebase, the most robust and cost-effective stack is **Vercel** + **Google Cloud Run**.

### 1. Backend (FastAPI) on Google Cloud Run
Google Cloud Run is "serverless," meaning it scales from zero to thousands of users automatically.
*   **Step 1:** Create a project in [Google Cloud Console](https://console.cloud.google.com/).
*   **Step 2:** Install the [Google Cloud SDK](https://cloud.google.com/sdk/docs/install) on your machine.
*   **Step 3:** Run the following commands in the `/backend` folder:
    ```bash
    gcloud auth login
    gcloud run deploy izra-api --source . --region us-central1 --allow-unauthenticated
    ```
*   **Step 4:** Note the URL provided (e.g., `https://izra-api-xyz.a.run.app`).

### 2. Frontend (Next.js) on Vercel
Vercel is the creator of Next.js and provides the best optimization.
*   **Step 1:** Push your code to a GitHub/GitLab/Bitbucket repository.
*   **Step 2:** Import the repo into [Vercel](https://vercel.com/).
*   **Step 3:** Configure Environment Variables in Vercel:
    *   `NEXT_PUBLIC_API_URL`: Set this to your Cloud Run URL.
*   **Step 4:** Deploy. Vercel will handle the SSL (HTTPS) and Global CDN.

## 📱 Phase 2: Publishing to Play Store (APK/AAB)

Since Izra AI is already a high-quality PWA, the best way to hit the Play Store is using **Trusted Web Activity (TWA)** via **Bubblewrap**.

### 1. Preparation
Ensure your live website is running on **HTTPS** (Vercel handles this).

### 2. Generate the Android App
We use [Bubblewrap](https://github.com/GoogleChromeLabs/bubblewrap), the official Google tool for converting PWAs to Play Store apps.

*   **Step 1: Install Bubblewrap CLI**
    ```bash
    npm install -g @bubblewrap/cli
    ```
*   **Step 2: Initialize the App Project**
    ```bash
    bubblewrap init --manifest=https://your-domain.com/manifest.json
    ```
    *   Follow the prompts to set your **App Name**, **App ID** (e.g., `com.izra.ai`), and **KeyStore** details.
*   **Step 3: Build the Signed AAB (Android App Bundle)**
    ```bash
    bubblewrap build
    ```
    *   This generates an `.aab` file (required for Play Store) and a `.apk` file (for direct testing).

---

## 🔒 Phase 3: Linking the App to the Website

Google requires you to prove ownership of the website to allow the app to run in "Trusted" mode (removing the URL bar).

1.  **Generate Asset Links:** Bubblewrap will provide a `assetlinks.json` file.
2.  **Upload to Website:** Place this file in your frontend under `public/.well-known/assetlinks.json`.
3.  **Verify:** Once deployed, Google will verify that `izra.ai` and your App ID are linked.

---

## 🎖️ Phase 4: Play Store Listing

1.  Go to [Google Play Console](https://play.google.com/console/).
2.  Create a new app and fill in the store listing (descriptions, screenshots).
3.  Upload the `.aab` file generated by Bubblewrap.
4.  Submit for review!

### 💡 Why this approach?
*   **Single Codebase:** Any change you make to the website automatically updates the app (no need to resubmit to Play Store for UI changes).
*   **Speed:** Bubblewrap setup takes < 15 minutes.
*   **Scalability:** Cloud Run and Vercel handle heavy loads effortlessly.
